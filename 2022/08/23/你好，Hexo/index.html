洛耶的伪教学

你好 Hexo(*^▽^*)

关于独立博客的事，在没接触的时候是非常向往的。既可以不受平台限制，畅所欲言，又可以自己编辑个性化的页面，实现个人美化自由。一路上总是充满了各种困难，在克服困难后也能长舒一口气，这就是我对搭建博客初体验的感受。就用这篇生活记录来作为我在Hexo的第一篇博客吧！

搭建博客记录

Gitee注册

为什么把这个放前面呢，因为要想把自己的博客通过互联网分享给别人，就需要Pages服务

目前Gitee和Github都有对应的仓库服务，这里讲一下Gitee（具体原因详见about）

简单进行用户注册后，登录工作台 - Gitee.com简单新建一个仓库。然后到设置里面把个人实名认证做了，不然没有服务权限。这个审核是真的费时间的emmm，我们可以继续搭建博客以等待审核。

Git、Node.js安装

具体安装路径

  • 从这里git-scm.com下载Git。

    安装后会发现右键菜单多了这两个选项。

    里面的Git Bash Here会在你鼠标所在的位置充当CMD控制台，如果想保持桌面整洁可以在任意地方新建一个文件夹进行后续安装工作。以后我们会经常用git命令做事。

  • 从这里nodejs.org下载Node.js。

    这个就是给JavaScript一个适用的环境，毕竟我们搭建web网页不能只局限于一动不动的静态web，具有动态效果和交互效果才是我们追求的动态web。

接下来在刚刚的Git Bash（以下简称git）里通过查询版本测试一下安装是否成功

git --version

node -v

npm -v

安装Hexo

首选进官网，看教程Hexo，具体步骤如下。

1、在git里面输入指令

npm install -g hexo-cli

2、初始化博客环境

初始化的命令格式为 hexo init <项目名称>

这里按照需求，将项目名称写作blog，不可随意更改。

hexo init blog

3、新建完成后，指定文件夹目录下有以下内容，大致了解一下：

node_modules: 依赖包 暂时可以不管

public：存放生成的页面 

scaffolds：给出文章会用到的模板

source：用来存放你的文章

themes：主题，也可以称作搭建博客的框架，非常实用

config.yml: 博客的配置文件，重点需要我们
预览博客

还记得我们的右键吗，初始化完成之后，就在blog文件夹下输入下面的指令

hexo s

就可以访问 http://localhost:4000/查看你的博客，开始写作了。

后续写博客及入库工作

1、本地启动项目

（s 表示 server ）

hexo s/server     #启动服务预览 

hexo s -p 端口号   #重置端口号

如：

hexo server -p 5000 

hexo server -i 192.168.1.1 #自定义 IP
2、创建新的博客
hexo n [layout] <title>

如：

hexo new "欢迎你，新来的旅客" 

hexo new page "pageName" #新建页面

*layout 表示文章布局，可选（post page draft）

默认使用 _config.yml 中的 default_layout 参数代替（post）

*title 表示文章标题，如果标题包含空格的话，请使用引号括起来。

3、基本配置修改

Hexo站点的基本设置主要包括：

首页标题、子标题、简介、关键词（英文,隔开）、作者、 语言和时区

这些都需要修改，这样才算自己的网站嘛。

# Site 
title: 洛耶的龟龟洞天

subtitle: '直接开卷' 

description: '记录思考时刻、java 笔记，由 Hexo+Gitee 搭建。' 

keywords: 生活趣事,java见解,问题+答复

author: 洛耶

language: zh-CN 

timezone: 'Asia/Jiangsu'

主要参数如下

title: # 标题
subtitle: # 副标题
description: # 描述
keywords: # 关键字
author: # 作者
language: # 语言
timezone: # 时区
url: # 网址
root: # 根目录
permalink: # 文章链接格式
permalink_defaults: # 链接默认值
source_dir: # 源文件夹，内容的存储位置
public_dir: # 公用文件夹，静态文件的生成位置
tag_dir: # 标签目录
archive_dir: # 存档目录
category_dir: # 分类目录
skip_render: # 复制到原始路径，不进行渲染
new_post_name: # 新帖子的文件名格式
titlecase: # 将标题转换为小写1/大写2 *
external_link.enable: # 在新标签页中打开外部链接
post_asset_folder: # 启用资源文件夹功能
filename_case: # 将文件名转换小写小写1/大写2
relative_link: # 是否创建相对于根文件夹的链接
index_generator.per_page: # 每页显示的文章数
index_generator.order_by: # 发布顺序
date_format: # 日期格式
time_format: # 时间格式
per_page: # 每个页面上显示的文章数
pagination_dir: # 网址格式
theme: # 主题名称
theme_config: # 主题配置，覆盖主题默认值
deploy: # 部署设置
include: # 包括隐藏文件
exclude: # 排除文件/文件夹
ignore: # 忽略文件/文件夹

重启下 Hexo 服务，我们刷新页面，就会发现上面的修改生效啦～
